Adicionando Dependências

Com exceção de programas muito simples, a maioria dos projetos Java utiliza APIs externas. Essas APIs, que normalmente estão no formato JAR, são chamadas de dependências e devem estar disponíveis para que os programas possam utilizá-las. Em projetos que não utilizam nenhuma ferramenta de build, essas APIs geralmente são armazenadas em um diretório separado e são referenciadas na compilação dos códigos-fonte. Nos projetos Maven, como era de se esperar, todas as dependências são declaradas no arquivo POM.

Para entender a sintaxe da declaração de uma dependência, vamos utilizar como exemplo o JUnit, o framework de testes que já vem como dependência padrão quando criamos um projeto.

<project ...>
...
--<dependencies>
----<dependency>
------<groupId>junit</groupId>
------<artifactId>junit</artifactId>
------<version>4.12</version>
----</dependency>
----<dependency>
------<!--outra dependência-->
----</dependency>
----...
--</dependencies>
...
</project>

Para cada dependência, precisamos especificar três informações: groupId, artifactId e version. Sua suspeita sobre toda dependência ser também um projeto Maven está correta. Quando declaramos uma dependência e executamos um build, o Maven vai ao repositório central, procura por um projeto com as três coordenadas e faz o download para o repositório local(/home/username/.m2/repository). Na próxima vez em que executar o build, como o artefato já está no repositório local, o Maven não precisará mais fazer o download.
