Adicionando os Javados no Site de um Projeto

Nos últimos dois posts, vimos como gerar as páginas do site de um projeto com o plugin Site e como gerar os Javadocs das classes com o plugin Javadoc. Neste tutorial, vamos utilizar os dois plugins em conjunto para adicionar a documentação das classes no site do projeto.

Por conveniência, vamos utilizar o mesmo projeto do último post, o string-int-math, que possui os comentários Javadoc na classe e nos métodos. Vamos alterar apenas o arquivo POM e configurar os plugins de forma que, quando invocarmos a fase site, os javadocs estejam disponíveis no site do projeto.

---arquivo pom.xml---

Para que o plugin Site inclua a documentação na página web do projeto, precisamos adicionar o plugin Javadoc na seção reporting. Como não queremos gerar os javadocs da classe de teste, configuramos somente o report javadoc na seção reports. Se nenhum report for especificado, a documentação das classes de teste é gerada por padrão.

Agora só precisamos invocar a fase site e abrir o arquivo index.html( localizado no diretório target/site) no browser para ver o resultado.

$ mvn site

--- arquivo index.html no browser ---

Ao clicar em Project Reports, obtemos uma lista de todos os reports gerados, onde há um link para os javadocs do projeto. Esse link nada mais é do que uma referência ao arquivo index.html do diretório target/site/api-docs.
