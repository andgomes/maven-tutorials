Gerando o Arquivo Distribuível de um Projeto

Uma das operações mais comuns do Maven é o packaging de um projeto, essa operação gera um arquivo distribuível que contém todas as classes e todos os arquivos de recurso. O formato do arquivo distribuível depende do package do projeto, definido na tag packaging do arquivo POM. Neste tutorial, assim como temos feito nos tutoriais anteriores, vamos utilizar o package JAR para gerar um arquivo neste formato.

Vamos criar um projeto chamado simple-project e invocar a fase package. Lembre-se de que, quando criamos um projeto com o archetype quickstart, o packaging já é definido como JAR por padrão.

$ mvn package

O arquivo JAR será criado e salvo no diretório target. O nome do arquivo segue a sintaxe [artifactId]-[version].jar. Por exemplo, se o artifactId do nosso projeto é simple-project e a versão 1.0, o nome do arquivo JAR será simple-project-1.0.jar. Além do arquivo distribuível, são criados os diretórios correspondentes às fases anteriores ao package(como compile e test) e um arquivo chamado pom.properties, localizado no diretório maven-archiver.
