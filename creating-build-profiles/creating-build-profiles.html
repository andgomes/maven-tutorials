Criando Profiles

Em algumas situações, precisamos executar builds que sejam portáveis em plataformas ou ambientes específicos. Para isso, precisamos que o build detecte em qual plataforma está executando e tome as decisões corretamente. Por exemplo, um projeto que utiliza bibliotecas de sistema deve saber qual função chamar para não correr o risco de, estando no Windows, chamar uma função do Linux. Para resolver esse problema, o Maven permite a criação de profiles que podem ser ativados se determinadas condições forem satisfeitas, de forma que cada profile tenha suas próprias configurações.

O Maven permite a criação de três tipos de profiles. A diferença entre os tipos está relacionada ao escopo de cada um.

* Profiles de projeto são definidos no arquivo POM e só estão disponíveis naquele projeto específico.
* Profiles de usuário são específicos para cada usuário e são definidos no arquivo /home/username/.m2/settings.xml.
* Profiles globais são acessíveis por qualquer usuário do ambiente e são definidos no arquivo M3_HOME/conf/settings.xml.

Como exemplo, vamos criar um profile simples de projeto com o id dev.

<project ...>
--...
--<profiles>
----<profile>
------<id>dev</id>
------<activation>
--------<activeByDefault>false</activeByDefault>
------</activation>
------</profile>
--</profiles>
</project>

No corpo da tag <activation>, podemos colocar várias condições de ativação do profile, como sistema operacional, versão do JDK, properties, etc. No nosso caso, vamos apenas definir o profile como desativado configurando para false o valor de <activeByDefault>.
