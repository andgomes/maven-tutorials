Compilando com o Plugin Compiler do Maven

Uma das primeiras ações a executar depois de termos escrito o código de uma classe Java é compilá-lo. Neste tutorial irei mostrar como compilar os códigos-fonte do diretório main/java/ com o plugin Compiler do Maven.

Antes de executar o comando para compilar, é interessante saber que, por padrão, o Compiler assume que o seu código-fonte está escrito de acordo com a versão 1.5 do Java, ou seja, se seu código utilizar lambda expressions(da versão 1.8) ou o diamond operator(da versão 1.7), o plugin Compiler lançará um erro informando que seu código não é compatível com a versão 1.5.

Para alterar a versão do código-fonte(source) e a versão do arquivo compilado(target), altere o arquivo pom.xml de acordo com a versão do Java que a sua instalação do Maven está utilizando. Por exemplo

<project ...>
--...
--<build>
----<plugins>
------<plugin>
--------<groupId>org.apache.maven.plugins</groupId>
--------<artifactId>maven-compiler-plugin</artifactId>
--------<version>3.5.1</version>
--------<configuration>
----------<source>1.7</source>
----------<target>1.7</target>
--------</configuration>
------</plugin>
----</plugins>
--</build>
--...
</project>

Depois de configurar a versão do source e do target, você já pode compilar seu código-fonte. Para isso digite o comando

$ mvn compile

O Maven irá salvar os arquivos .class no diretório target/classes/, seguindo a mesma estrutura de diretórios dos arquivos de código fonte.

target/
--- classes/
--- --- com/
--- --- --- andgomes/
--- --- --- --- App.class
