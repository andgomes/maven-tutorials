Gerando a Página Web de um Projeto

Um dos diferenciais do Maven é a possibilidade de utilizar plugins para os mais diversos tipos de funcionalidades, que vão além de builds relacionados somente com a aplicação em si. Por esse motivo, o Maven não é considerado apenas uma ferramenta de build, mas uma ferramenta de gerenciamento de projetos. Neste tutorial, veremos como gerar uma página HTML com a documentação básica de um projeto Maven através do plugin Site, o plugin que é utilizado quando invocamos a fase site do lifecycle Site.

Vamos criar um projeto simples com o artifactId simple-project utilizando o archetype Quickstart. Além disso, vamos adicionar a descrição do projeto na tag description do arquivo POM. Essa descrição será exibida na página web gerada pelo plugin.

$ mvn archetype:generate -DgroupId=org.andgomes -DartifactId=simple-project -DarchetypeArtifactId=maven-archetype-quickstart -DinteractiveMode=false

pom.xml

<project xmlns="http://maven.apache.org/POM/4.0.0"
---------xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
---------xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
-----------------------------http://maven.apache.org/maven-v4_0_0.xsd">
--<modelVersion>4.0.0</modelVersion>
--<groupId>org.andgomes</groupId>
--<artifactId>simple-project</artifactId>
--<packaging>jar</packaging>
--<version>1.0-SNAPSHOT</version>
--<name>simple-project</name>
--<url>http://maven.apache.org</url>
--<description>
----Projeto simples criado pelo archetype Quickstart
--</description>
--<build>
----<plugins>
------<plugin>
--------<groupId>org.apache.maven.plugins</groupId>
--------<artifactId>maven-project-info-reports-plugin</artifactId>
--------<version>2.7</version>
------</plugin>
----</plugins>
--</build>
--<dependencies>
----<dependency>
------<groupId>junit</groupId>
------<artifactId>junit</artifactId>
------<version>4.12</version>
------<scope>test</scope>
----</dependency>
--</dependencies>
</project>

Perceba que na seção plugins foram especificadas as coordenadas do plugin Project Info Reports. O motivo é que, dependendo da versão desse plugin, são adicionadas mais páginas ao site do projeto e, ao incluir a versão mais recente, podemos ter um site com mais opções disponíveis. Além disso, se o plugin não for incluído no arquivo POM, uma mensagem de warning aparecerá durante o build recomendando que uma versão seja especificada.

Ao invocar a fase site, o plugin Site irá gerar os arquivos HTML do nosso projeto.

$ mvn site

Se o build for executado corretamente, o diretório site será criado como subdiretório de target. É nesse diretório que os arquivos HTML são armazenados.

pom.xml src/ | main/java/org/andgomes/App.java | test/java/org/andgomes/AppTest.java target/ | site/ | | css/ | | images/ | | index.html | | project-info.html | | project-summary.html | | ...

Cada arquivo HTML corresponde à uma seção do site do projeto. Vamos abrir o arquivo index.html no browser e ver o resultado.
