Copiando Arquivos de Recurso com o Plugin Resources

Quando programas precisam de uma quantidade muito grande de informação como entrada ou precisam acessar propriedades pré-definidas, é comum armazenarmos essas informações em arquivos separados. Esses arquivos são comumente chamados de recursos(resources).

Em muitos projetos esses arquivos são salvos em um diretório chamado res. Na estrutura de projetos do Maven, no entanto, há dois diretórios nos quais podemos armazenar os recursos, dependendo se são recursos dos programas de produção ou dos programas de teste.

Os recursos dos programas de produção devem ser salvos no diretório src/main/resources/. Da mesma forma, os recursos dos programas de teste devem ser salvos no diretório src/test/resources/.

Vamos criar um projeto e salvar um arquivo de recurso chamado resource.txt no diretório src/main/resources. A estrutura do projeto será como a estrutura abaixo.

src/
----main/
--------java/
------------com/
----------------andgomes/
--------------------App.java
--------resources/
------------resource.txt
----test/
--------java/
------------com/
----------------andgomes/
--------------------AppTest.java
pom.xml

Agora vamos invocar a fase process-resources com o comando

$ mvn process-resources

Ao invocar a fase process-resources, o Maven utiliza o goal resources do plugin Resources para copiar os arquivos de recurso. O arquivo resource.txt é copiado para o diretório target/classes/, conforme a estrutura abaixo

src/
----main/
--------java/
------------com/
----------------andgomes/
--------------------App.java
--------resources/
------------resource.txt
----test/
--------java/
------------com/
----------------andgomes/
--------------------AppTest.java
target/
----classes/
--------resource.txt
pom.xml

Da mesma forma, podemos copiar os arquivos de recurso das classes de teste com o comando

$ mvn process-test-resources

O Maven utiliza o goal testResources do plugin Resources para copiar os arquivos de recurso para o diretório target/test-classes/.

Perceba que não é tão útil ter somente os arquivos de recurso no diretório das classes. Ao invés de invocar a fase process-resources, o mais comum é invocar a fase compile, por que além de compilar o código-fonte, lembre-se que a fase process-resources já é invocada implicitamente pelo Maven no lifecycle default, ou seja, somente invocando a fase compile, copiamos os recursos e compilamos o código.
