Os Ciclos de Vida do Maven

Depois de termos visto algumas operações básicas nos posts anteriores, resolvi separar esse post para falar sobre uma parte mais conceitual do Maven. Iremos ver a estrutura do Maven e o passo-a-passo da execução dos comandos, apresentando mais detalhadamente alguns termos técnicos, como lifecycles, phases e goals.

A primeira coisa a saber é que há ciclos de vida(lifecycles) associados aos comandos do Maven que temos executado até agora. Os lifecycles são divididos em três: default, clean e site. Esses ciclos de vida são divididos em fases(phases), e são essas fases que invocamos quando queremos executar alguma operação. Por exemplo, o comando

$ mvn compile

invoca a fase compile do lifecycle default. Algumas fases dos três lifecycles são listadas abaixo.

Default Lifecycle
initialize: prepara o build configurando propriedades, criando diretórios, etc.
process-resources: copia e processa os resources para o diretório das classes compiladas.
compile: compila os códigos-fonte de produção.
test-compile: compila os códigos-fonte de teste.
test: executa os testes e cria os relatórios.
package: empacota os artefatos do build em um arquivo que pode ser distribuível, como um JAR, por exemplo.
install: salva no repositório local o arquivo distribuível gerado na fase package.

Clean Lifecycle
pre-clean: executa ações necessárias antes da limpeza do target.
clean: remove o diretório target gerado no build anterior.

Site Lifecycle
site: cria as páginas web do projeto.
site-deploy: faz o deploy(implantação) das páginas web geradas na fase site.

Quando executamos uma fase, todas as fases anteriores do lifecycle são executadas. Por exemplo, quando invocamos a fase compile, antes de executar os goals dessa fase, o Maven executa as fases initialize e process-resources.

O principal objetivo da separação do build em fases é garantir a execução de passos anteriores necessários à fase que o usuário quer executar. Quem realmente executa as operações são os plugins. Cada tipo de packaging associa certos plugins às fases do lifecycle. Por exemplo, se o tipo do packaging for jar, quando invocamos a fase compile, o Maven executa o goal compile do plugin Compiler para compilar o código-fonte. Da mesma forma, quando invocamos a fase clean é o goal clean do plugin Clean que remove o diretório target.

Ao invés de invocarmos uma fase e essa fase executar um goal, podemos executar o goal diretamente, basta informar o nome do plugin e nome do goal que queremos executar. Por exemplo, para executar o goal compile do plugin Compiler, podemos executar o comando

$ mvn compiler:compile

É importante notar que, ao executar o goal diretamente, o Maven não invocou a fase process-resources, que é uma fase anterior à fase compile. Isso não aconteceu porque não invocamos uma fase, e nesse caso o lifecycle não é referenciado. Portanto, quando queremos que todas as operações-padrão do lifecycle sejam executadas antes de uma operação específica, devemos invocar a fase ao invés de invocar o goal diretamente.
