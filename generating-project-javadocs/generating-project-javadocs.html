Gerando os Javadocs de um Projeto

Em projetos com muitas classes, navegar pelos arquivos Java geralmente não é a solução mais eficiente para entender ou explicar a estrutura da aplicação e o relacionamento entre seus componentes. Quando se trata de um projeto Java, a ferramenta javadoc nos permite gerar páginas HTML contendo a documentação das classes do projeto automaticamente, construindo essa documentação a partir de comentários que inserimos nos arquivos Java. Esses comentários são conhecidos como comentários javadoc e são delimitados pelas strings "/**" e "*/". Neste tutorial, veremos como gerar os javadocs de um projeto Maven utilizando a ferramenta javadoc através do plugin Javadoc.

Como exemplo, vamos utilizar o projeto string-int-math, um projeto simples que contém uma classe utilitária, uma classe de exceção e uma classe de teste. Faça o download e extraia o projeto. Segue abaixo alguns fragmentos das classes e do arquivo POM.

<project ...>
--...
--<groupId>org.andgomes</groupId>
--<artifactId>string-int-math</artifactId>
--<packaging>jar</packaging>
--<version>0.0.1</version>
--<name>String-Int Math</name>
--...
--<build>
----<plugins>
------<plugin>
--------<groupId>org.apache.maven.plugins</groupId>
--------<artifactId>maven-javadoc-plugin</artifactId>
--------<version>2.10.4</version>
------</plugin>
----</plugins>
--</build>
--<dependencies>
----<dependency>
------<groupId>junit</groupId>
------<artifactId>junit</artifactId>
------<version>4.12</version>
------<scope>test</scope>
----</dependency>
--</dependencies>
</project>

---StringIntMathTest.java---

No arquivo POM, adicionamos o plugin Javadoc na seção plugins. A classe StringIntMath possui comentários Javadoc para a descrição da classe e dos métodos. Esses comentários serão inseridos na página da documentação pelo plugin Javadoc ao executarmos o goal javadoc.

$ mvn javadoc:javadoc

As páginas da documentação serão criadas e armazenadas no diretório target/site/api-docs. O diretório string-math-api deve resultar na seguinte estrutura:

pom.xml
src/
--main/java/org/andgomes/
----stringmath/
------StringIntMath.java
------exception/
--------InvalidStringNumberException.java
--test/java/org/andgomes/stringmath/StringIntMathTest.java
--target/
----javadoc-bundle-options/
----site/apidocs/
------org/andgomes/stringmath/
------resources/
------index.html
------package-list
------allclasses-frame.html
------...

Vamos abrir o arquivo index.html no browser e ver o resultado. Esse arquivo contém o Overview do projeto, com o nome(definido na tag <name>) e uma lista dos packages.

---index.html no browser---

Clicando na classe StringIntMath na seção All Classes no canto esquerdo, podemos ver a documentação da classe com a descrição, o nome do autor e a versão, além descrição de cada método na seção Method Summary.
