Gerando os Javadocs de um Projeto

Em projetos com muitas classes, navegar pelos arquivos Java geralmente não é a solução mais eficiente para entender ou explicar a estrutura da aplicação e o relacionamento entre seus componentes. Quando se trata de um projeto Java, a ferramenta javadoc nos permite gerar páginas HTML contendo a documentação das classes do projeto automaticamente, construindo essa documentação a partir de comentários que inserimos nos arquivos das classes. Esses comentários são conhecidos como comentários javadoc e são delimitados pelas strings "/**" e "*/". Neste tutorial, veremos como gerar os javadocs de um projeto Maven utilizando a ferramenta javadoc através do plugin Javadoc.

Como exemplo, vamos utilizar o projeto string-int-math, um projeto simples que contém uma classe utilitária, uma classe de exceção e uma classe de teste. Faça o download e extraia o projeto. Segue abaixo alguns fragmentos das classes e do arquivo POM.

< >

<project ...>
--...
--<groupId>org.andgomes</groupId>
--<artifactId>string-int-math</artifactId>
--<packaging>jar</packaging>
--<version>0.0.1</version>
--<name>String-Int Math</name>
--...
--<build>
----<plugins>
------<plugin>
--------<groupId>org.apache.maven.plugins</groupId>
--------<artifactId>maven-javadoc-plugin</artifactId>
--------<version>2.10.4</version>
------</plugin>
----</plugins>
--</build>
--<dependencies>
----<dependency>
------<groupId>junit</groupId>
------<artifactId>junit</artifactId>
------<version>4.12</version>
------<scope>test</scope>
----</dependency>
--</dependencies>
</project>
