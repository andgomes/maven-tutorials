Testes são essenciais no desenvolvimento de softwares com complexidade média ou alta. Possuir um ambiente onde a execução de testes seja facilitada é fundamental para qualquer programador. Afinal, se a execução de teste leva muito tempo para ser configurada, a probabilidade dessa parte do projeto ser negligenciada pelos programadores é alta.

O Maven facilita a execução de testes automatizados através do plugin Surefire, e é o goal test desse plugin que usaremos para executar e gerar o relatório dos testes. O plugin Surefire nos dá a flexibilidade de executar testes com os frameworks JUnit e TestNG, mas neste tutorial irei utilizar exemplos com o JUnit.

Antes de criarmos uma classe de teste, vamos modificar a versão do JUnit no arquivo POM na seção de dependências para a versão 4.12. Isso simplificará nossas classes de teste, pois utilizaremos as anotações do Java.

<project ...>
--...
--<dependencies>
----...
----<dependency>
------<groupId>junit</groupId>
------<artifactId>junit</artifactId>
------<version>4.12</version>
------<scope>test</scope>
----</dependency>
----...
--</dependencies>
--...
</project>

Agora vamos criar um teste bem simples, só para utilizarmos como exemplo. Iremos chamar a classe de TestAssert e salvar o arquivo no diretório /src/java/test/com/andgomes.

package com.andgomes;

import org.junit.Assert.assertTrue;

public class TestAssert {
----@Test
----public void testAssertTrue() {
--------assertTrue(true);
----}
}

Execute o comando para executar o teste

$ mvn test
