Tutorial de Instalação do Apache Maven no GNU/Linux

Neste tutorial, irei apresentar os passos necessários para instalar a última versão do Apache Maven(atualmente, a 3.3). Como o Maven é baseado em Java, o único pré-requisito é ter o JDK instalado no computador.

obs: alguns comandos deste tutorial só são aceitos com o modo sudo.

Há pelo menos duas formas possíveis de instalar o Maven: através do programa apt do GNU/Linux ou configurando a variável de ambiente PATH. Com o programa apt, só há um passo necessário: executar o comando apt-get install mvn. Porém, nem sempre a versão do repositório é a mais recente e o usuário não tem tanto controle sobre a localização dos diretórios do Maven, ao contrário da instalação manual, que foi a que eu escolhi para esse tutorial.

Antes de qualquer coisa, temos que fazer o download do Maven no site oficial da Apache Foundation.

Na seção Files, clique no link do arquivo apache-maven-3.3.9-bin.tar.gz e aguarde o download.

Para extrair o arquivo tar.gz, primeiro abra o Terminal e entre no diretório onde o arquivo foi baixado com o comando cd diretório-do-arquivo. Por exemplo, se o arquivo estiver no diretório /home/username/Downloads, o comando será cd /home/username/Downloads.

Em seguida, execute o comando tar -zxvf apache-maven-3.3.9-bin.tar.gz. Se a operação terminar corretamente, o diretório apache-maven-3.3.9 será extraído para o diretório atual. Pode ser necessário instalar o programa tar, que pode ser instalado através do utilitário apt do GNU/Linux com o comando apt-get install tar.

Depois de extraído, o ideal é que o diretório seja movido para um local mais apropriado que o diretório de Downloads. Além disso, você pode querer mudar o nome do diretório do Maven para um nome mais curto, por exemplo, apenas maven. Para isso, escolha o diretório destino e mova o diretório do Maven com o comando mv apache-maven-3.3.9/ /opt/maven/, onde /opt/maven/ é o diretório de destino que escolhi.

Agora temos que configurar a variável de ambiente PATH do GNU/Linux para que, ao digitarmos os comandos do Maven, o sistema saiba onde procurar o arquivo de execução. Para isso, vamos criar a variável M3_HOME com o comando export M3_HOME=/opt/maven, e incluir essa variável na variável PATH com o comando export PATH=$M3_HOME/bin:$PATH. Depois de alterar e exportar a variável PATH já podemos começar a usar o Maven. Porém, toda vez que você iniciar uma sessão(logar) terá que digitar os comandos novamente. Uma forma de evitar isso é salvar o comando em algum arquivo de inicialização do GNU/Linux. Tanto o arquivo /etc/profile quanto o /home/user/.profile servem ao nosso propósito, com apenas uma diferença, o /home/username/.profile só fica disponível para o seu usuário, diferentemente do /etc/profile, que fica disponível para todos os usuários do computador. Sabendo disso, abra um dos dois arquivos, copie os comandos export M3_HOME=/opt/maven e export PATH=$M3_HOME/bin:$PATH ao final e depois salve o arquivo. Para a alteração fazer efeito é necessário reiniciar a sessão.

Para testar a instalação, basta digitar o comando mvn -v ou mvn --version. Se tudo estiver correto, o resultado do comando deve ser algo parecido com o resultado abaixo.
