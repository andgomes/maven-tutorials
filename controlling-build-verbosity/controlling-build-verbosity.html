Controlando a Verbosidade do Build

Em algumas situações, podemos querer aumentar ou diminuir a quantidade de informação que build do Maven exibe no terminal. Existem três opções que podem ser utilizadas para controlar a verbosidade do build.

-q ou --quiet

Com a opção -q, o Maven só exibe informações do build se houver algum erro durante a execução. Se o build finalizar com sucesso, nada é exibido. Em verdade, se os testes forem executados, o relatório ainda será mostrado no terminal.

-e ou --errors

A opção -e habilita o output de erros nos plugins. Se algum erro ocorrer, o stack trace detalhado do erro no plugin será exibido.

-X ou --debug

A opção -X faz com que um extenso log seja exibido. Essa opção é útil para desenvolvedores de plugins.
