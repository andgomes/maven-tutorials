Controlando a Verbosidade do Build

Em algumas situações podemos querer aumentar ou diminuir a quantidade de informações que build do Maven exibe no terminal. Existem três opções que podem ser utilizadas para controlar a verbosidade do build.

-q ou --quiet

A opção -q faz com que o build não mostre nada se tudo ocorrer bem. Somente erros são mostrados.

-e ou --errors

A opção -e habilita o output de erros nos plugins. Se algum erro ocorrer, o stack trace detalhado do erro no plugin será exibido.

-X ou --debug

A opção -X faz com que um extenso log seja exibido. Essa opção é útil para desenvolvedores de plugins.
